# 📱 Регламент разработки мобильного приложения

Данный документ описывает порядок выполнения задач, тестирования и выпуска мобильного приложения.

---

## 1. Выполнение задач (тасков)

- Все задачи ведутся в системе управления проектами (сейчас в Telegram - 'Ошибки App').
- Каждая задача должна иметь:
  - Описание (по хорошему дополнить данные аккаунта и устройства например: 'Iphone 7 15 iOS, не авторизованный ...')
  - Приоритет
- После завершения разработки:
  - Выполняется self-review (ревью своего кода)
  - Выполняется сборка приложения

---

## 2. Первичное тестирование

После завершения разработки задачи выполняется первичное тестирование на следующих устройствах:

### iOS:
- iPhone 7 (iOS 15)
- iPhone 14 (iOS 18)
- iPhone 17 (iOS 26)
- iPhone 16 Pro Max (iOS 26) физически

### Android:
- эмулятор Android API 36 (с актуальной версии Android) или Google Pixel 2
- Samsung Galaxy S23

### Проверяется:
- Корректность UI
- Работоспособность бизнес-логики
- Отсутствие критических багов

Если обнаружены баги — исправление и повторное тестирование на данном этапе.

---

## 3. Публикация тестовой версии

После успешного прохождения первичного тестирования:

- Сборка загружается в:
  - Google Play (внутреннее тестирование)
  - TestFlight (iOS тестирование)
- Передается в чат, что выпущена новая версия для тестирования
- Написать список изменений/правок

### 3.1 Обнаружение багов тестировщиками

Если тестировщики выявляют баги:
- Создаются задачи на исправление
- Выполняется доработка
- Процесс повторяется с пункта **2 (Первичное тестирование)**

---

## 4. Расширенное тестирование

После успешного прохождения тестирования тестировщиками проводится дополнительное тестирование и проходится по всем тест-кейсам на следующих устройствах:

### iOS:
- iPhone SE
- iPhone 11
- iPhone 12
- iPhone 13
- iPhone 14
- iPhone 15
- iPhone 16
- iPhone 17
(версии без Pro) - все симуляторы

### Android:
- Google Pixel 2
- Samsung Galaxy A50
- Samsung Galaxy S23 - физическое устройство
- Xiaomi Redmi Note 7
(в дальнейшем дополнить список, сейчас тест в Genymotion, дальше подключить Browserstack и прийти к написанию тестов разделов)

### Проверяется:
- Адаптивность интерфейса
- Производительность на слабых устройствах
- Совместимость версий ОС
- Работоспособность проекта

---

## 5. Выпуск в Production

После успешного прохождения трех этапов тестирования:

- Обновляется версия приложения
- Формируются финальные Release Notes
- Выполняется публикация:
  - App Store
  - RuStore
  - Google Play
- Производится мониторинг:
  - Краш-репортов (Hawk + Firebase)
  - Логов (Hawk + Firebase)
  - Отзывов пользователей (на платформах)

---

# 🔁 Общая схема процесса

Разработка → Первичное тестирование → TestFlight / Google Play тест →
Доработка (при необходимости) → Расширенное тестирование → Production

---

# 📌 Важно

- Ни одна версия не может быть выпущена в Production без прохождения всех этапов тестирования.
- Все баги фиксируются в системе управления задачами.
- Критические баги устраняются в первую очередь.
